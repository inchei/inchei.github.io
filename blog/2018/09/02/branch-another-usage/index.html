分支的編外用法

發佈於 Sep 2, 2018

廢話

指以``.``開頭的配置文件。

話説很久以前我在 GitHub 上創建了一個倉庫（repository）叫做「層曡樣式表存檔（CSS-Archieves）」，本就是儲藏寫得太多的樣式表。折騰起 WSL 之後對於 Linux 的配置方式感到非常新奇，便把 dotfiles 也塞進了此倉庫中。經歷了多次改名，最後順著潮流直接叫做「dotfiles」了，寓意并非 dotfiles，硬要扯的話，應是其引申義「配置文件」。

然而順著潮流之後，令人苦惱的事情發生了—— GitHub 所顯示的語言并不是人見人愛的 shell 或者 vimscript，而是看上去便不太令人愉快的 CSS。

怎麽辦呢？我首先想到的是.gitattributes，便大筆一揮——

這真是個糟糕的主意，因爲 GitHub 上的高亮系統會就照著.gitattributes去做，使得無辜的 CSS 面對高亮手足無措。

分支（Branch）

分支在 GitHub 中是用來幹什麽的呢？

我大約是可憐兮兮地查詢了根本一次也沒有看過的GitHub Help ，其如是説：

You can use branches to:

Develop features

Fix bugs

Safely experiment with new ideas

想必大部分以 GitHub 鬧著玩（折騰玩具項目）的同志，見到這樣的描述便會認定：分支和我一點關係都沒有。

腦洞

最終，我覺得 CSS 和 dotfiles 這個稍微有點高級的詞匯實在不太搭得上邊，便思索新建一個分支存儲 CSS 文件。

驚喜發生了！

指奔走相告，此乃本文真實目的。

原來 GitHub 的倉庫語言會直接無視非主分支下的内容，此困擾我許久的問題終於解決，喜大普奔！

只恐怕世上并沒有和我持相同問題的人😂。
